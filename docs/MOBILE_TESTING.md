# 📱 移动端测试指南

## 测试环境

### 浏览器开发者工具测试
1. 打开 Chrome/Firefox 开发者工具 (F12)
2. 点击设备工具栏图标 (Ctrl+Shift+M)
3. 选择设备或自定义尺寸：
   - iPhone 12/13/14: 390×844
   - iPhone SE: 375×667
   - Samsung Galaxy S20: 360×800
   - iPad: 768×1024

### 真实设备测试
- iOS Safari
- Android Chrome
- 微信内置浏览器

## 常见问题修复

### 问题1: 内容被底部导航栏遮挡
**解决**: 已为所有页面添加 `padding-bottom: 6rem`

### 问题2: iOS输入框自动缩放
**解决**: 输入框字体大小设置为 `16px`

### 问题3: 按钮太小难以点击
**解决**: 所有按钮最小高度 `44px` 或 `48px`

### 问题4: 横屏显示异常
**解决**: 已添加横屏优化样式

## 测试命令

### 启动开发服务器
```powershell
cd frontend
npm run dev
```

### 在移动设备上测试
1. 确保手机和电脑在同一网络
2. 查看开发服务器输出的 Network 地址（如 `http://192.168.x.x:3000`）
3. 在手机浏览器访问该地址

## 响应式断点

- **移动端**: ≤768px
- **小屏移动端**: ≤480px
- **平板**: 769px - 1023px
- **桌面**: ≥1024px
